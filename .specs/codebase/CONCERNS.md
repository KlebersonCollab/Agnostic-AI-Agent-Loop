# Critical Risks & Concerns

> Most important section for the Safety Valve protocol. Every concern cites file evidence. Generated by the SDD Explorer.

## Technical Debt
1. **~~Manual dual-registry synchronization~~ — RESOLVED** — `tools/__init__.py` previously required editing both `TOOLS_METADATA` and `TOOLS_MAP` by hand. It now defines a single `REGISTERED_TOOLS` list of `(ToolDefinition, handler)` tuples; `TOOLS_METADATA` and `TOOLS_MAP` are derived from it, and import-time validation raises `ImportError` on duplicate names or non-callable handlers. *Risk: eliminated for tool additions.*
2. **`AgentListener` is a plain class, not an ABC** — `agent.py:18`. Subclasses may forget required overrides; no enforcement. Documented as intentional (README) but still a latent contract risk.
3. **No `[build-system]` / `[project.scripts]`** — `pyproject.toml`. Project relies on `uv` + `python main.py`; not pip-installable as a console command. *Risk: low, but limits distribution.*
4. **MCP configs unused** — `.agents/mcp/chrome-devtools.json`, `.agents/mcp/playwrite.json` exist but are never imported/referenced by any Python module. *Risk: dead config / confusion.*

## Fragile Areas / Test Gaps
1. **`GeminiProvider` & `AnthropicProvider` untested in isolation** — `tests/test_providers.py` only covers `openai`, `openai_compatible`, `openrouter` branches of `get_provider`. The `_generate()` logic for Gemini/Anthropic (message normalization, tool-call synthesis) has no unit coverage. *Evidence: README "Não coberto" section; `tests/test_providers.py`.*
2. **`cli.py` / `run_cli()` untested** — argument parsing, checkpoint detection/resume, and provider bootstrap are exercised only manually or via the real benchmark. *Evidence: README coverage gaps.*
3. **`multi_agent.py` flow untested** — `spawn_subagents_parallel`, `CollectingAgentListener`, and subagent loop only run via `test_benchmark_real_calls.py` (real API, auto-skipped). *Evidence: README coverage gaps.*
4. **`retry_with_backoff` untested** — the permanent-vs-transient error classification and backoff timing in `providers/base.py` has no direct test. *Risk: retry behavior could regress silently.*
5. **`ContextBuilder` frontmatter parser is hand-rolled** — `context_builder.py:_parse_frontmatter` does naive line-splitting; nested YAML or quoted values with `:` may misparse. No YAML library used. *Risk: medium for complex skill metadata.*

## Security & Performance
1. **`calculate` uses `eval`** — `tools/math_tools.py`. Mitigated by char whitelist + `__builtins__=None`, but `eval` remains a sensitive primitive; acceptable given the strict allowlist.
2. **Path-traversal guard is prefix-based** — `io_tools.py` checks `abs_path.startswith(cwd)`. Symlinks or `..` resolution edge cases could bypass; relies on `os.path.abspath` only. *Risk: low-medium.*
3. **`search_grep` caps at 100 results** — `io_tools.py:search_grep` truncates output; large codebases may hide matches beyond the cap. Acceptable for context-size control.
4. **Subagent context sharing** — `spawn_subagents_parallel` reuses the single active provider across threads (`multi_agent.py`). Provider clients (OpenAI/Gemini/Anthropic SDKs) are generally thread-safe, but concurrent token usage / rate limits are not managed. *Risk: medium under many parallel subagents.*
5. **`checkpoint.json` written to project root** — `agent.py` writes full conversation history to `checkpoint.json` (gitignored). If gitignore is misconfigured, history (potentially sensitive) could be committed. *Risk: low (gitignored).*

## Mandatory Paths (do not modify without review)
- `tools/__init__.py` — `REGISTERED_TOOLS` single-source registry + import-time validation; `TOOLS_METADATA`/`TOOLS_MAP` derivation.
- `agent.py` — ReAct loop, tool interception, handover checkpoint logic.
- `providers/base.py` — `BaseLLMProvider` contract + `retry_with_backoff`.
- `context_builder.py` — dynamic prompt compilation (affects every LLM call).
