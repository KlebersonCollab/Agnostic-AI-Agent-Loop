## Style & Typing
- **Type hints:** Used throughout (function signatures annotated with `List`, `Dict`, `Optional`, `Any`, `Set`).
- **Docstrings:** Present on public functions/classes (Google-ish style, e.g. `list_project_files`, `read_file`, `patch_file`).
- **Imports:** Standard library first, then third-party (`providers`, `tools`), then local. Lazy imports inside methods for heavy SDKs (`from openai import OpenAI` inside `__init__`, `from google.genai import types` inside `_generate`).
- **Error handling:** Tools return **strings** (not raise) with `"Error: ..."` prefixes; the agent loop wraps tool execution in try/except and returns error strings (`agent/core.py`). Provider `generate()` uses `retry_with_backoff` (`providers/base.py`) distinguishing permanent vs transient errors.
- **Thread safety:** File writes/edits guarded by `threading.Lock` (`_file_write_lock` in `tools/io_tools.py`); subagents run via `ThreadPoolExecutor`; `AgentMemory` uses a class-level `threading.RLock`.
- **Security:** FS tools resolve `os.path.abspath` and reject paths outside `os.getcwd()` via `pathlib.Path.relative_to` (not `startswith`) (`read_file`, `write_file`, `patch_file`, `get_outline`). `calculate` restricts `eval` with `__builtins__=None`.

## Testing Conventions (evidence from `tests/`)
- Framework: `pytest`. Files named `test_*.py` in `tests/` (18 files: `test_agent`, `test_agent_orchestrator`, `test_async_delegation`, `test_benchmark_real_calls`, `test_cli_loop`, `test_concurrent_tool_execution`, `test_context_breakdown`, `test_context_builder`, `test_context_references`, `test_curl_tool`, `test_hooks`, `test_io_tools`, `test_math_tools`, `test_mcp_client`, `test_memory`, `test_memory_graph`, `test_new_tools`, `test_providers`).
- Tests use **Mock providers** subclassing `BaseLLMProvider` (e.g. `MockProvider` in `test_agent.py`) to avoid real API calls.
- Assertions use `pytest.raises` for validation (e.g. `test_providers.py::test_get_provider_factory_validation`).
- `monkeypatch.setenv` used to inject API keys for provider init tests.
- Real-call benchmark (`test_benchmark_real_calls.py`) is auto-skipped without a valid `OPENROUTER_API_KEY`.
- No fixtures file / `conftest.py` present; tests are self-contained with inline fixtures.

## Commenting
- Sparse inline comments, mostly explaining non-obvious logic (e.g. Anthropic alternating-message merge in `providers/anthropic.py:_convert_messages`, Gemini synthetic tool-call IDs in `providers/gemini.py`).
- No TODO/FIXME/HACK markers found in source code (only in `.agents/rules` and skill docs as policy text).
