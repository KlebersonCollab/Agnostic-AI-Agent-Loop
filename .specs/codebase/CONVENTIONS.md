# Core Conventions

> Naming and style conventions extracted from codebase evidence. Generated by the SDD Explorer.

## Naming
- **Files:** `snake_case.py` modules (`agent.py`, `cli.py`, `context_builder.py`); packages `providers/`, `tools/`, `tests/`.
- **Classes:** `PascalCase` — `Agent`, `AgentListener`, `ContextBuilder`, `BaseLLMProvider`, `OpenAIProvider`, `ConsoleAgentListener`, `CollectingAgentListener`, `ToolDefinition`, `ChatMessage`, `ToolCall`, `MessageRole`.
- **Functions/methods:** `snake_case` — `run_cli`, `build_prompt`, `get_provider`, `spawn_subagents_parallel`, `list_project_files`, `read_file`, `write_file`, `patch_file`, `search_grep`, `get_outline`, `calculate`, `set_active_provider`.
- **Module-level constants:** `UPPER_SNAKE` — `SYSTEM_PROMPT` (`agent.py`), `_file_write_lock` (`io_tools.py`), `_active_provider` (`multi_agent.py`), `TOOLS_METADATA`, `TOOLS_MAP`.
- **Provider classes:** `<Vendor>Provider` pattern (`OpenAIProvider`, `GeminiProvider`, `AnthropicProvider`, `OpenRouterProvider`, `OpenAICompatibleProvider`).
- **Skills/Rules:** `.agents/skills/<name>/SKILL.md` (frontmatter `name`/`description`/`keywords`); `.agents/rules/<Name>.md` (active by default).

## Style & Typing
- **Type hints:** Used throughout (function signatures annotated with `List`, `Dict`, `Optional`, `Any`, `Set`).
- **Docstrings:** Present on most public functions/classes (Google-ish style, e.g. `list_project_files`, `read_file`, `patch_file`).
- **Imports:** Standard library first, then third-party (`providers`, `tools`), then local. Lazy imports inside methods for heavy SDKs (`from openai import OpenAI` inside `__init__`, `from google.genai import types` inside `_generate`).
- **Error handling:** Tools return **strings** (not raise) with `"Error: ..."` prefixes; the agent loop wraps tool execution in try/except and returns error strings (`agent.py:177-180`). Provider `generate()` uses `retry_with_backoff` (`providers/base.py`) distinguishing permanent vs transient errors.
- **Thread safety:** File writes/edits guarded by `threading.Lock` (`_file_write_lock` in `io_tools.py`); subagents run via `ThreadPoolExecutor`.
- **Security:** FS tools resolve `os.path.abspath` and reject paths outside `os.getcwd()` (`read_file`, `write_file`, `patch_file`, `get_outline`). `calculate` restricts `eval` to digits/operators with `__builtins__=None`.

## Testing Conventions (evidence from `tests/`)
- Framework: `pytest`. Files named `test_*.py` in `tests/`.
- Tests use **Mock providers** subclassing `BaseLLMProvider` (e.g. `MockProvider` in `test_agent.py`) to avoid real API calls.
- Assertions use `pytest.raises` for validation (e.g. `test_providers.py::test_get_provider_factory_validation`).
- `monkeypatch.setenv` used to inject API keys for provider init tests.
- Real-call benchmark (`test_benchmark_real_calls.py`) is auto-skipped without a valid `OPENROUTER_API_KEY`.
- No fixtures file / `conftest.py` present; tests are self-contained.

## Commenting
- Sparse inline comments, mostly explaining non-obvious logic (e.g. Anthropic alternating-message merge in `anthropic.py:_convert_messages`, Gemini synthetic tool-call IDs in `gemini.py`).
- No TODO/FIXME/HACK markers found in source code (only in `.agents/rules` and skill docs as policy text).
